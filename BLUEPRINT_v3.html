1.2 The Outlook email — how it works without IT setup

Web apps cannot open Outlook with an attachment via mailto:. The reliable method: the system generates a .eml draft file — clicking it opens Outlook with To / Subject / body already filled and the Excel file already attached; the user just presses Send. Works with desktop Outlook and the system email general.anca@gmail.com profile. (Optional later upgrade: Microsoft Graph API to send silently — needs IT to register an Azure app; the .eml route needs nothing.)

  • Request email (auto, on submit): To = the staff's L1 approver (falls to L2/L3 on later steps); Subject "[ขออนุมัติสั่งซื้อ] PO-2606-001 — {name} ({dept})"; body = bilingual summary table; attachment = PO-2606-001.xlsx.
  • Supplier email (admin, after approval): To = supplier contact from the supplier register; Subject "ขอใบเสนอราคา / Request for quotation — AMS PO-2606-001"; the same Excel attached, trimmed to that supplier's lines.

1.4 Approval lines — separate from Leave/OT, admin-configurable

Unlike Leave/OT (which reads the Mgr column automatically), purchase approval lines are often budget-based, so they get their own table po_approval_lines (emp_no, level1_emp_no, level2_emp_no, level3_emp_no), editable per staff in System Admin and bulk-uploadable via Excel (columns: Emp No., L1, L2, L3 — blank = skip level). If a staff has no line configured, it falls back to the Mgr-column chain.

2 · Multi-supplier design (your point 6)

A supplier register table: name (OfficeMate, Makro, Lotus's, HardwareHouse, …), contact email(s), default lead-time, payment terms, active flag. Each catalog product gets a supplier_key + the catalog import screen gains a supplier selector, so one catalog holds many suppliers side by side. Image handling per supplier: OFM = CDN URL by code (already works); others = optional image upload or URL column in the import file.

7.2 Anti-cheat video player (YouTube embed)

  • YouTube IFrame API: seek/fast-forward disabled (jump ahead → snapped back to furthest watched second), speed locked to 1×
  • Watch-time heartbeat every 10 s; pause on tab blur (switch window → video pauses, timer stops)
  • Test button unlocks only at ≥ 95% genuine watch-time of EVERY video; slides/photos require paging to the last page (each page logs a view)
  • Honest limit: browser-side enforcement can't stop someone pointing a phone at the screen — but it fully blocks skip/speed/inactive-tab cheating, which is the realistic goal.

8 · Document printing v2 (ISO 14001 paper records)

  • Exact-format guarantee: สปส.1-03 and ล.ย.01 move from "reconstructed layout" to overlay-on-original: the real government form is the background image and the system prints the data at calibrated positions — the printout files identically to a hand-filled original. The job application (FM-HR-003) + the 20260612 additional document merge into ONE print set fed by ONE fill-in (the candidate enters data once).
  • Template versioning: each printable form is a template record (file + field-position map + version + effective date). When a standard changes you upload the new template version — old printouts stay reproducible, new prints use the new version. No code edit.
  • Direct-print popup: "🖨️ Print" opens the filled document in a popup that calls window.print() immediately (same pattern as the org-chart print built today) — no download-then-open step.
  • Integrity statement before submit (candidate portal): bilingual declaration the candidate must tick, e.g. "ข้าพเจ้าขอรับรองว่าข้อความข้างต้นเป็นความจริงทุกประการ หากปรากฏภายหลัง ว่าข้าพเจ้าปกปิดข้อเท็จจริงหรือให้ข้อความอันเป็นเท็จ ข้าพเจ้ายินยอมให้บริษัทเลิกจ้างได้ทันทีโดยไม่ จ่ายค่าชดเชย ตามพระราชบัญญัติคุ้มครองแรงงาน พ.ศ. 2541 มาตรา 119(1)" — exact legal wording verified at build time (s.119(1) covers dishonest acts / intentional false statements; criminal-law threats are avoided as they could be unenforceable and intimidating).
  • Completion checklist: admin screen listing every employee × required document fields, % complete, missing-item drill-down, Excel export — tracks the paper-record readiness per ISO 14001.
